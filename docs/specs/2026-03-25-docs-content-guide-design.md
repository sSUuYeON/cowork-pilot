# docs/ 콘텐츠 작성 가이드 — 설계 문서

> 메타 에이전트가 생성하는 docs/ 파일의 내용 품질을 일관되게 유지하기 위한 작성 가이드 시스템
> 작성일: 2026-03-25
> 상태: Draft

---

## 1. 목표

메타 에이전트(Phase 3)의 스캐폴더가 생성하는 빈 템플릿 파일들에 에이전트가 내용을 채울 때, 품질과 형식이 일관되도록 가이드를 제공하는 시스템.

문제:
- 현재 Jinja2 템플릿에는 섹션 헤더만 있고 "(Phase 2 하네스가 채울 영역)"으로 비어있음
- 에이전트마다 테이블/리스트/코드블록 사용이 달라져 일관성 없음
- 분량 기준이 없어서 한 줄로 끝내거나 소설을 쓸 수 있음

해결:
- **접근 C**: 공통 규칙은 AGENTS.md의 `## Writing Standards` 섹션에, 문서 타입 특화 가이드는 각 .j2 템플릿에 HTML 주석으로 인라인
- Cowork 세션은 AGENTS.md를 항상 먼저 읽으므로 공통 규칙 참조 누락 없음

---

## 2. 가이드 배치 아키텍처

```
AGENTS.md
  └── ## Writing Standards          ← 공통 규칙 (형식/분량/금지사항)
        ↓ (Cowork가 자동으로 읽음)

각 생성된 .md 파일
  └── <!-- GUIDE: ... -->           ← 문서 타입 특화 가이드 (인라인)
        ↓ (에이전트가 채운 후 삭제)
```

가이드 삭제 규칙: 에이전트가 섹션 내용을 채우면 해당 `<!-- GUIDE: ... -->` 주석을 삭제한다. Chunk 5 크로스 링크 검증에서 남아있는 GUIDE 주석이 있으면 미완료로 판단.

적용 범위: 이 가이드는 **스캐폴더가 생성하는 빈 템플릿 파일**에만 적용된다. `docs/project-brief.md`는 사용자가 AI와 대화로 채우는 입력 문서이므로 이 가이드의 대상이 아니다.

분량 기준 해석: 모든 분량 범위(예: "5~15줄")는 **가이드라인**이다. 정당한 이유가 있으면 범위를 벗어날 수 있으나, 최소값 미만은 불완전한 것으로 간주한다. 최대값 초과는 허용하되 2배를 넘기지 않는다.

## 3. AGENTS.md 공통 작성 규칙

AGENTS.md.j2 템플릿에 `## Writing Standards` 섹션으로 포함. 스캐폴딩이 생성하는 모든 프로젝트의 AGENTS.md에 자동 포함됨.

```markdown
## Writing Standards

이 프로젝트의 모든 docs/ 파일은 다음 규칙을 따른다.

### 형식 규칙
- 테이블: GitHub Flavored Markdown 테이블 사용. 3컬럼 이하 권장
- 리스트: 정의가 필요하면 `- **항목**: 설명` 형식. 나열만이면 `- 항목`
- 코드: 인라인은 백틱, 블록은 언어 명시 코드 펜스
- 다이어그램: ASCII art 또는 Mermaid. 5줄 이하면 ASCII, 그 이상이면 Mermaid

### 분량 규칙
- 각 섹션: 최소 3줄, 최대 30줄 (코드블록/테이블 제외)
- 한 줄짜리 섹션 금지 — 최소한 "무엇을, 왜, 어떻게" 3가지를 다뤄야 함
- 빈 섹션 금지 — 해당 없으면 "해당 없음 — (이유)" 한 줄 작성

### 금지 사항
- "추후 작성", "TBD", "TODO" 단독 사용 금지 — 대신 현재 알고 있는 것이라도 적을 것
- 브리프 내용 그대로 복붙 금지 — 반드시 구체화/확장해서 작성
- 주관적 표현 ("좋은", "적절한") 금지 — 측정 가능한 기준으로 대체

### GUIDE 주석 처리
- 각 파일의 `<!-- GUIDE: ... -->` 주석은 해당 섹션의 작성 가이드
- 내용을 채운 후 반드시 GUIDE 주석 삭제
- GUIDE 주석이 남아있으면 해당 섹션 미완료로 간주
```

### 4.1 ARCHITECTURE.md

## 1. 시스템 개요

## 2. 기술 스택

-->

## 3. 핵심 설계 결정
<!-- GUIDE:
- 내용: 아키텍처 레벨 결정. 각각 결정/근거/버린 대안 포함

## 4. 디렉토리 구조
<!-- GUIDE:

- 형식: 트리 구조 코드블록 + 각 항목 1줄 설명
- 분량: 10~25줄
-->

## 5. 데이터 흐름
<!-- GUIDE:
- 내용: 핵심 사용자 시나리오 1~2개의 요청→응답 흐름

### 4.4 도메인 문서 — auth.md, deployment.md 등 (design-docs/)
